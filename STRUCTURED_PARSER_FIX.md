# 结构化格式解析器修复总结

## 问题描述

TypeScript 版本的剧本解析器缺少对结构化格式的支持，导致以下问题：
1. 一行中多个 `（TypeX・角色名）内容` 模式无法拆分
2. `O1：选项文本（$var，跳转）` 格式无法识别
3. `O1 分支【O1｜场景】` 分支声明无法处理
4. `（跳转 Oend）` 跳转指令无法处理
5. `【Start｜场景描述】` 标签+场景格式无法处理

## 修复内容

### 1. 扩展类型定义 (`types.ts`)

新增结构化格式节点类型：
- `struct_label`: 标签+场景（如 `【Start｜场景描述】`）
- `struct_option`: 结构化选项（如 `O1：选项（$var，跳转）`）
- `struct_branch`: 分支声明（如 `O1 分支【O1｜场景】`）
- `struct_jump`: 跳转指令（如 `（跳转 Oend）`）

### 2. 增强分类器 (`classifier.ts`)

- 添加结构化格式正则表达式
- 优先检测结构化格式模式
- 正确解析选项的变量和跳转目标

### 3. 预处理优化 (`parser.ts`)

- 添加 `splitTypedDialogues` 函数
- 自动拆分一行中的多个 `（TypeX・name）` 模式
- 这是 Python 版本也没有的功能改进

### 4. 后处理完善 (`postProcess.ts`)

- 处理结构化格式节点
- 将选项附加到前面对话行
- 处理跳转指令，添加到前一行 commands
- 为分支起始行添加变量指令

## 与 Python 版本对比

| 功能 | Python 版本 | TypeScript 版本 |
|------|------------|----------------|
| 多对话模式拆分 | ❌ 不支持 | ✅ 支持 |
| 结构化选项 | ✅ 支持 | ✅ 支持 |
| 分支声明 | ✅ 支持 | ✅ 支持 |
| 跳转指令 | ✅ 支持 | ✅ 支持 |
| 标签+场景 | ✅ 支持 | ✅ 支持 |

**TypeScript 版本的改进**：自动拆分一行中的多个 `（TypeX・name）` 模式，这是 Python 版本没有的功能。

## 使用建议

结构化格式要求每条指令独占一行：
- ✅ 正确：每个 `（TypeX・name）内容` 单独一行
- ✅ 正确：每个选项 `O1：文本（变量，跳转）` 单独一行
- ✅ 正确：分支声明 `O1 分支【O1｜场景】` 单独一行
- ⚠️ 注意：跳转指令 `（跳转 Oend）` 会附加到前一行
